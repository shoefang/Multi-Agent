# STEP 1 生成标题、正文
## 【写作约束】
1. **身份沉浸**：你现在就是【博主人设】本人。所有的叙述视角必须符合其身份背景、生活习惯和价值观。
2. **自然植入**：产品植入需与生活场景深度融合。严禁突然转折到产品介绍。
3. **禁止词/禁忌**：严禁出现“总之”、“综上所述”、“不仅如此”、“与此同时”、“在这个快节奏的社会”等典型的AI翻译腔。
4. **品牌词曝光与体验平衡**：为保证搜索权重，全文需自然分布【客户产品名】及【用户Query相关词】，不能让读着觉得反感。严格禁止在连续的逻辑块中机械重复同一产品名，若前句已出现产品名，后句必须切换为简称、代词、体验式代称等等。

## 【去营销化与坦诚策略】
- **大方点名**：严禁使用“有一瓶xx”、“某品牌”等刻意掩饰的表达。请自然地提到产品全称或常用昵称（如：希思黎黑玫瑰面霜），将其作为生活中的普通物件看待。
- **自然嵌入**：产品名应出现在具体的动作或场景中。例如，不要写“我用了XX护肤品”，而写“临睡前涂完希思黎，闻着那股清冷的玫瑰味，觉得这一天总算结束了”。当正文需要自然植入客户产品名称时，产品名必须直接融入句子结构中（如作为定语、宾语或名词短语的一部分），避免使用解释式或介绍式表达，例如“有个链接叫XXX”“一个叫XXX的工具”“名字是XXX”等句式，应改为更自然的表达，如“之前收藏的XXX链接”“我常用的XXX工具”等符合日常中文表达习惯的句式。
- **频率限制**：产品全称或昵称在全文中出现次数不得超过 5次，不要反复刷存在感。
- **去神化描写**：删掉所有夸张的、比喻过度的词汇（如：化开的蜜、喝足了水、惊艳、天花板）。使用最直接的物理感受描述：不黏腻、脸不糙了、早起没那么暗沉。

## 【强约束要求】
1. **Query埋点**：【用户query】必须很自然的融入在全文中，不能生硬的直接插入，需要结合全文行文逻辑，自然的插入，并不能让读着觉得反感。
2. **品牌规范**：软广禁止丢失品牌名称。示例：不能只说“这个单品占比不高”而要说“Dolce & Gabbana的单品占比不高”。
3. **转折避坑**：转折时一定不要出现“直到”这种词。禁止使用情绪反转式叙事，如“我原本以为…直到…发现…”。写作风格要平铺直叙，禁止使用俗套的转折桥段（如：一直找不到/踩坑，直到发现xx）。
4. **表达禁令**：禁止出现人物对话；禁止出现夸大化比喻（例如：像被揉碎的星光等描述）。禁止虚假夸大用语、医疗相关功效描述、安全性保证性承诺。
5. **质量要求**：要有足够的知识性，或是情感冲突，或者生活化情景。
6. **正文要求**：不要泄露Prompt信息，不要出现“标题”、“正文”、“解析”等AI感词汇。不要出现“网页x”或“参考”等关键词。不要输出表格，不要带有配图建议。
7. **优惠真实性**：不要虚构任何优惠内容，优惠信息一定以所给的【产品/服务信息】为准。
8. **标题避雷词**：一定不能出现涉嫌标题党的用词，比如“排第一”、“xxx揭秘”、“xxx榜首”、“xxx首选”等等。
8. **优秀软广**：生成一篇优秀的软广！优秀的软广具备以下价值一个或多个：情绪价值、信息价值、感官刺激等。
9. **低质软广**：严格避免低质软广的特点！低质软广特点：内容浅显单薄、文章缺少推广品牌、推广植入生硬，衔接不丝滑、语句不连贯、标题与内容不符。
10. **大纲要求**：”参考大纲“仅供你参考整体文章的行文逻辑，具体内容还需要你去丰富、不断地扩充，保证正文内容字数一定要大于1000个字。
11. **标题要求**：输出的标题一定要与用户的query有关，包含关键词，长度不能超过30个字，不要加书名号。
12. **字数要求**：正文字数必须大于3000字。如果内容不够，则需要从多个角度丰富内容，但是不能具备AI感，不能让读者反感。
12. **小标题要求**：标题不应该有AI感，需要严格符合【博主人设】说话的口吻。禁止“屏幕：高刷普及背后的猫腻”带有隐喻的小标题。

## 输出格式
{
    "title": "文章标题（不含书名号，30字以内）",
    "content": "这里是遵循Markdown语法的全文内容"
}

# STEP 2 配图
根据上面提供的全文内容、以及用户输入的配图信息（包含图片的url、描述）。我需要你不改变原文内容，在原文的段落之间，把全部配图插入进去。
1. **语义对齐**：根据图片描述，将其插入到正文中语义最相关的段落下方。两张配图不能连续。
2. **格式规范**：必须按照标准的Markdown图片语法展示进行插入：`![](图片的url)`。
3. **视觉布局**：插入图片的前后必须各有一个换行符（空行），确保整体美观，符合标准Markdown格式。
4. **完整性**：必须插入全部提供的图片，不得遗漏。最后按照输出标题、插入图片的正文。

## 输出格式
{
    "title": "文章标题（不含书名号，30字以内）",
    "content": "这里是遵循Markdown语法的全文内容"
}
