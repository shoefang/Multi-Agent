# STEP 1 生成标题、正文  

## 【写作约束】  
1. **身份沉浸**：你现在就是【博主人设】本人。所有的叙述视角必须符合其身份背景、生活习惯和价值观。  

## 【表达与风格要求】  
- **自然表达**：语言要贴近日常表达，避免刻意雕琢或生硬书面语。句式可以有长有短，但整体要流畅。  
- **细节优先**：多使用具体生活场景、细节描写来支撑观点，而不是空泛总结。  
- **情绪真实**：可以有情绪，但不夸张、不煽情，不过度渲染。  
- **避免套路**：不要使用模板化表达（如“很多人都不知道…”、“其实你一直都错了…”等）。  

## 【强约束要求】  
1. **Query埋点**：【用户query】必须自然融入全文语境中，不能生硬插入，需要结合叙事或论述自然出现。  
2. **逻辑完整**：文章需要有清晰的结构推进（如从经历→思考→总结，或问题→分析→解决），但不要显式写出结构提示。  
3. **转折自然**：避免生硬反转或刻意制造冲突，不使用“直到”这类强转折词。  
4. **表达禁令**：  
   - 禁止出现人物对话  
   - 禁止夸张比喻（如“像宇宙爆炸一样震撼”）  
   - 禁止虚假、极端、绝对化表述  

5. **内容质量**：文章需具备以下至少一种价值：  
   - 信息价值（知识、经验、方法）  
   - 情绪价值（共鸣、表达、思考）  
   - 观察价值（对生活/现象的洞察）  

6. **正文要求**：  
   - 不要出现“标题”“正文”“总结”等提示词  
   - 不要有明显AI生成痕迹  
   - 不引用“网页”“资料显示”等来源提示  
   - 不使用表格形式  

7. **大纲要求**：“参考大纲”仅作为结构参考，需充分扩展内容，保证内容丰富、有层次。  

8. **标题要求**：  
   - 必须与用户query相关，并包含核心关键词  
   - 长度不超过30字  
   - 风格自然，不标题党，不夸张  

9. **字数要求**：  
   - 正文字数必须大于1500字  
   - 若内容不足，需要从多个角度补充（经历、分析、延伸思考等），但保持自然  

## 【小标题要求】  
- 可以使用小标题，但要像真实作者写的，不要刻意总结式或模板式  
- 避免隐喻型标题或过度概括（如“人生的答案”“时代的选择”）  
- 小标题应服务内容，而不是形式感  

--------------------------------------------------  

## 输出格式
{
    "title": "文章标题（不含书名号，30字以内）",
    "content": "这里是遵循Markdown语法的全文内容"
}

# STEP 3 配图
根据上面提供的全文内容、以及用户输入的配图信息（包含图片的url、描述）。我需要你不改变原文内容，在原文的段落之间，把全部配图插入进去。
1. **语义对齐**：根据图片描述，将其插入到正文中语义最相关的段落下方。两张配图不能连续。
2. **格式规范**：必须按照标准的Markdown图片语法展示进行插入：`![](图片的url)`。
3. **视觉布局**：插入图片的前后必须各有一个换行符（空行），确保整体美观，符合标准Markdown格式。
4. **完整性**：必须插入全部提供的图片，不得遗漏。最后按照输出标题、插入图片的正文。

## 输出格式
{
    "title": "文章标题（不含书名号，30字以内）",
    "content": "这里是遵循Markdown语法的全文内容"
}
