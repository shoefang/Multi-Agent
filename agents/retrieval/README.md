# 自适应信息采集 Agent (DeepCollect)

一个基于大语言模型的智能信息采集系统，采用 **Skill + Tool Calling** 架构，通过编排器调度多个专职子 Agent 完成复杂的信息采集任务。

## 项目概述

本项目是一个**自适应信息采集Agent**，核心思想是：**根据查询复杂度动态调整检索深度和广度**。

系统会：
1. 首先评估用户查询的复杂程度（1-5级）
2. 根据复杂度决定检索轮次和策略
3. 每轮检索后评估信息增益，智能判断是否继续
4. 最终生成包含核心观点总结的深度分析报告

---

## 核心特性

### 1. Skill + Tool Calling 架构
- 主编排器通过 `run_sub_skill` 调度子 Agent
- 子 Agent 拥有独立上下文，职责清晰
- 通过 `complete_task` 正式退出，保证流程可控
- **极佳的 Token 优化与防幻觉设计**：主 Agent 仅保留任务骨架和各轮次高度浓缩的结论摘要（“会议纪要”模式）。子 Agent 读取数万字的网页原文提炼出数百字摘要后即刻销毁，避免了多轮复杂检索中海量无用文本堆积导致的 Token 浪费与模型“遗忘/幻觉”。

### 2. 复杂度自适应
- 自动评估查询复杂度（1-5级）
- 简单查询1-2轮快速完成，复杂查询5-10轮深度挖掘

### 3. 动态检索扩展
- **横向扩展（广度）**：扩展到相关概念、竞品对比等
- **纵向扩展（深度）**：深入分析细分维度

### 4. 智能停止判断
- 信息增益较低时提前停止
- 所有维度已覆盖时停止
- 多轮检索后增益下降时停止

### 5. 核心观点提炼
- 3句话核心结论
- 关键洞察揭示深层规律

│  │   │  Step 4: write_file + complete_task                   │ │ │

### 核心 API

```python
# 运行子 Skill
run_sub_skill(
    skill_name="01_复杂度评估",  # 子 Skill 名称
    context_json="{...}",        # 传给子 Agent 的上下文
    query="用户查询"             # 主任务 query
)

# 完成任务并退出
complete_task(
    summary="任务完成摘要",
    files_created=["./out/xxx/report.md"]
)

# 执行检索
search_docs(
    query="检索词",
    display=1,      # 权威来源数量
    undisplay=3     # UGC 来源数量
)
```

## 运行流程

### 详细流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    用户输入 Query                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Phase 1: 复杂度评估 (01_复杂度评估)                          │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  分析 query 复杂度，返回:                             │    │
│  │  - complexity_level (1-5级)                          │    │
│  │  - dimensions (核心维度列表)                          │    │
│  │  - max_rounds (建议最大轮次)                          │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Phase 2: 动态检索循环 (02_动态检索与增益评估)               │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  for round in 1..max_rounds:                          │   │
│  │                                                        │   │
│  │    Step 1: 生成本轮检索词                              │   │
│  │    ├── 第1轮: 使用原始 query                           │   │
│  │    └── 后续轮: 横向/纵向扩展                           │   │
│  │                                                        │   │
│  │    Step 2: 执行检索 (search_docs)                      │   │
│  │                                                        │   │
│  │    Step 3: 评估信息增益 (new_info_score)               │   │
│  │                                                        │   │
│  │    Step 4: 判断停止条件                                │   │
│  │    ├── new_info_score <= 0.55 ? → 停止                │   │
│  │    ├── missing_dimensions 为空 ? → 停止                │   │
│  │    ├── round >= 3 且 score <= 0.65 ? → 停止           │   │
│  │    └── 否则继续下一轮                                  │   │
│  │                                                        │   │
│  │    返回: round_summary, continue_search                │   │
│  │  end for                                               │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Phase 3: 报告生成 (03_报告生成)                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  根据各轮摘要生成深度分析报告:                         │    │
│  │  - 核心观点总结 (3句话结论 + 关键洞察)                 │    │
│  │  - 详细分析 (按维度展开)                              │    │
│  │  - 信息来源与可信度                                   │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 写入文件并完成任务                                  │
│  - write_file 保存报告                                       │
│  - complete_task 正式退出                                    │
└─────────────────────────────────────────────────────────────┘
```

### 复杂度评估标准

| 级别 | 描述 | 预期轮次 | 示例 |
|-----|------|---------|------|
| 1级 | 简单事实查询 | 1-2轮 | "今晚油价调整时间" |
| 2级 | 单维对比/概览 | 2-3轮 | "飞科和飞利浦哪个好" |
| 3级 | 多维对比/浅层分析 | 3-5轮 | "飞科vs飞利浦：性能、价格、评价对比" |
| 4级 | 系统性研究 | 5-7轮 | "电动剃须刀行业技术发展趋势" |
| 5级 | 深度多维研究 | 7-10轮 | "全球个护小家电市场格局与技术演进" |

## Skill 定义

### 主编排器 (自适应信息采集/SKILL.md)

```markdown
# 自适应信息采集 — 主编排器

你是一个**任务编排器**，通过 `run_sub_skill` 工具调度三个专职子 Agent：

## 执行流程（严格按顺序）

### Step 1: 复杂度评估
run_sub_skill(skill_name = "01_复杂度评估", ...)

### Step 2: 动态检索循环
while continue_search = true and round < max_rounds:
    run_sub_skill(skill_name = "02_动态检索与增益评估", ...)

### Step 3: 报告生成
run_sub_skill(skill_name = "03_报告生成", ...)

### Step 4: 完成任务
write_file + complete_task
```

### 子 Skill 职责

| Skill | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 01_复杂度评估 | 评估 query 复杂度 | `{query}` | `complexity_level, dimensions, max_rounds` |
| 02_动态检索与增益评估 | 执行单轮检索 | `query, round_number, dimensions, ...` | `round_summary, new_info_score, continue_search` |
| 03_报告生成 | 生成分析报告 | `query, all_round_summaries, ...` | `report_path, report_content` |

---

## 停止条件

`02_动态检索与增益评估` Skill 内部的停止条件：

### 强制停止
- `round_number >= max_rounds` → 已达到最大检索轮数

### 提前停止（满足任一即停止）
- `new_info_score <= 0.5` → 信息增益较低，继续检索价值有限
- `missing_dimensions` 为空 且 `new_info_score <= 0.7` → 所有维度已全覆盖
- `round_number >= 3` 且 `new_info_score <= 0.6` → 已进行多轮检索且信息增益下降

### 信息增益评分标准

| 分数范围 | 含义 |
|---------|------|
| 0.0-0.4 | 几乎没有新信息，大量重复或与上轮高度相似 |
| 0.4-0.6 | 少量新信息，有一定补充但价值有限 |
| 0.6-0.75 | 较多新信息，有价值的补充 |
| 0.75-1.0 | 大量新信息，重要发现 |

## 快速开始

### 环境要求

- Python 3.8+
- 依赖包：requests, tqdm

### 运行示例

```python
from process import DeepCollectAgent

# 创建 Agent
agent = DeepCollectAgent(
    model="deepseek-v3.2",  # 模型名称
    max_rounds=10           # 最大检索轮次（运行时会根据复杂度动态调整）
)

# 使用 Skill 模式执行（推荐）
query = "今晚油价会上调吗 上调多少"
report = agent.run_skill(
    query=query,
    max_steps=72,           # 最大交互步数（防失控）
    skill_name=None         # 使用默认编排器
)

# 查看执行结果（摘要）
print(report)

# 报告已保存至: ./out/{query}/report.md
```
