# STEP 1 创意生成
## 任务目标
生成 3 个差异化的内容切入点（Angles）。
## 创作标准
1. 确定用户query与客户的具体信息，深入了解用户query的核心需求。
2. 总结可以满足用户需求、且与账号人设信息和客户信息相契合的宣传点。
3. 根据客户产品思考产品的目标群体需求，构思如何通过软广的方式满足。
4. 必须生成优秀的软广，可以参考优秀的软广特质。优秀的软广需具备：情绪价值、信息价值或感官刺激。
5. 严禁直接套用客户的专业参数，必须转化为生活化的解决思路。
6. 创意角度必须差异化（例如：一个侧重情绪价值，一个侧重物理痛点解决）。
7. 创意角度必须是基于软广文章的，不存在任何视频脚本等。
8. 所有生成的创意切入点的title，都必须与用户Query：{query}高度相关（或者包含部分），从而保证文章能够被召回。

## 输出格式
{
    "angles": [
        {
            "type": "切入维度（如：逆向思维/干货拆解/沉浸故事）",
            "title": "拟定标题",
            "logic": "核心逻辑简述",
            "value_point": "带给读者的核心价值"
        }
    ]
}

--------------------------------------------------

# STEP 2 大纲生成
从 STEP 1 中选择最匹配 Query 的创意生成 Markdown 结构大纲。

## 【结构布局要求】
1. 根据用户query和创意切入点设计一个故事背景。
2. Query埋点：规划【用户query】在全文中的自然植入位置，确保不生硬。
3. 转折避坑：大纲逻辑中禁止出现情绪反转式叙事（如"原本以为…直到…"）。
4. 参数处理：客户信息中的专业参数说明，一定不能直接套用在文中，必须转化成真实的体感描述。
5. 表达禁令：禁止出现人物对话；禁止出现浮夸的比喻（例如：像被揉碎的星光等描述）。禁止虚假夸大用语、医疗功效描述、安全性保证性承诺。
6. 优惠真实性：不要虚构任何优惠内容，优惠信息一定以所给的【产品/服务信息】为准。
7. 品牌规范：禁止丢失品牌名称。示例：不能只说"这个单品占比不高"而要说"Dolce & Gabbana的单品占比不高"。
8. 小标题要求：标题不应该有AI感，需要严格符合【博主人设】说话的口吻。禁止“屏幕：高刷普及背后的猫腻”带有隐喻的小标题。

## 输出格式
{
    "outline": Markdown格式的详细大纲
}

--------------------------------------------------

# STEP 3 额外信息需求判断
根据创意大纲，判断是否需要检索外部信息或配图。
1. 信息检索需求：如果该创意需要一些额外的真实信息作为支撑，请明确给出检索query。
2. 配图检索需求：如果该创意需要一些配图作为补充，请判断需要配图大致数量并写入输出。

## 输出格式
{
    "需要检索": 是/否
    "检索query":（如果不需要检索则为空字符串）
    "需要配图": 是/否
    "配图数量": （如果不需要配图则为空字符串）
}

--------------------------------------------------

# STEP 4 最终输出与文件写入
根据Step 3 产出的内容，将最终结果写入指定路径文件中。

写入规则：
- 将检索与配图需求写入demand.json,格式为:
{
    "需要检索": true/false
    "检索query":（如果不需要检索则为空字符串）
    "需要配图": true/false
    "配图数量": （如果不需要配图则为空字符串）
}
- 大纲规划写入plan.md，格式如下：
# [标题]
[正文内容（Markdown格式）]
